# Web Production Deliverable Atlas

Status: promoted atlas seed.

Retention policy: keep.

## Purpose

This atlas maps deliverables to audience, lifecycle, release relevance, and
retention policy.

## Deliverable Classes

```txt
source:
  user notes, source docs, code, assets, external references

generated:
  published mirrors, validation reports, generated reports, derived outputs

phase evidence:
  Phase 0/1/2/3/5 docs, review notes, close docs

handoff:
  NEXT-SESSION, CURRENT-STATE, ROADMAP, CHANGELOG

repository orientation:
  README.md, contribution/project docs

WordPress release:
  readme.txt, screenshot.png, style.css headers, theme.json, templates,
  patterns, license/notice package evidence

portfolio/stakeholder:
  final report, presentation narrative, process summary
```

## Audience Boundaries

```txt
developer/collaborator:
  README.md, corpus, atlas, core, phase docs

reviewer/future cycle:
  Phase 5 close, handoff meta-docs, routed-forward lists

WordPress.org:
  readme.txt, screenshot.png, theme package, review evidence

portfolio/stakeholder:
  final presentation/report
```

## Release-Readiness Rule

Theme Review requirements are valid references before release, but they become
active submission gates only when a distributable package exists.
