# Web Production Workflow Atlas

Status: promoted atlas seed.

Retention policy: keep.

## Purpose

This atlas maps reusable web-production workflow concepts from the refined
webdesign corpus to Axismundi cycle practice.

## Workflow Map

```txt
intake:
  source brief, user note, reference inventory, copyright/source confidence

concept:
  project vocabulary, ideation, scope, WBS-like work-package split

prototype:
  fidelity, visualization mode, storyboard, pilot candidate

IA:
  content structure, labeling, navigation model, page hierarchy

wireframe:
  page regions, content priority, interaction assumptions

visual_design:
  grouping, hierarchy, type/color/layout lens

responsive:
  layout stress, breakpoint basis, responsive pattern

usability:
  heuristic, UX dimension, accessibility gate

implementation:
  Pilot/distributable/WordPress surface, dependency and validation policy

handoff_report:
  deliverable set, retention policy, release-readiness, maintenance
```

## Axismundi Phase Mapping

```txt
initiating -> user GO / trigger / entry fence
planning -> Phase 0
research and analysis -> Phase 1
synthesis and execution -> Phase 2
monitoring and controlling -> Phase 3 + review/amend
closing -> Phase 5 + commit/push + memory/handoff
```

## Guardrails

```txt
Workflow notes do not directly authorize page layout.
TT5 and Google Sites must be evaluated through matrix rows.
Pilot probe findings do not automatically become distributable decisions.
Retention policy applies to workflow artifacts as well as release artifacts.
```
