# Reference Implementations

Validation targets, NOT distributable products. These exist to demonstrate that the binding ontology, design system, and platform ontology operationalize together.

Contents change to match the latest binding/ontology state. Reference implementations may be torn down and regenerated when the underlying ontology shifts.

- `theme-pilot/` — Axismundi Pilot v0.1 (WordPress block theme, v2.1a-P0.5 audit target)

For installable, semver-versioned products, see `../distributables/`.
