Pattern-driven templates

Twenty Twenty-Five keeps the outer template sparse and lets patterns carry the visible body. The index template is a short chain: header, constrained main, hidden blog heading, query loop, and footer.

This specimen keeps that decision intact. The post body is shown as full content, not as an excerpt, because the default TT5 blog templates are designed for complete reading flow.

May 26, 2026

One-column query loop

The source query loop uses a single post column with a 3:2 featured image, x-large linked title, full post content, and a small linked date. There are no cards and no masonry grid in the default index route.

That restraint matters for ontology capture: the layout role is fallback blog reading, not magazine browsing or portfolio scanning.

May 25, 2026

Footer as template part

The footer pattern combines site identity, tagline, two vertical navigation groups, and a small credit row. In this lab translation it remains structurally plain, so the template hierarchy stays readable.

Future Pilot work can map the same structure into block patterns without copying TT5 visual tokens, palette values, or font choices.
