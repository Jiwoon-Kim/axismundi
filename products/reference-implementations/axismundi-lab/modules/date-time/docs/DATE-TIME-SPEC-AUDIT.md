# Date/Time Picker Spec Audit

Status: v3.6.12 Route A completion
Module: `modules/date-time/`
Rows: Date picker #22 + Time picker #23

## Scope

Date+Time remains one combined lab module. v3.6.12 completes the bounded Date
grid a11y surface that was routed to BACKLOG #19 and records the current Time
picker contract without redesigning it.

## Date Picker Contract

The Date picker owns:

- single-date and range modes
- calendar, month, year, and input views
- roving date-cell focus
- Date grid keyboard navigation
- current-date and selected-date state exposure
- polite month/year announcements
- OK / Cancel commit model
- forbidden-ancestor initialization bail-out

The Date grid uses:

- `role="grid"` on the grid host
- `role="row"` wrappers generated by the runtime
- `role="gridcell"` on generated date buttons
- `aria-labelledby` from the grid to the month/year label
- `aria-current="date"` on the demo "today" cell
- `aria-selected` on selected start/end cells
- `aria-multiselectable="true"` only in range mode

Keyboard contract:

- ArrowLeft / ArrowRight: previous / next visible date
- ArrowUp / ArrowDown: previous / next week
- Home / End: start / end of the current week row
- PageUp / PageDown: previous / next month
- Shift+PageUp / Shift+PageDown: previous / next year
- Enter / Space: activate focused date through native button activation
- Escape: close picker

## Time Picker Contract

The Time picker keeps the current v3.6.12 contract:

- panel host: `role="dialog" aria-modal="false"`
- dial host: `role="listbox"`
- generated dial options: `role="option"` + `aria-selected`
- hour/minute and format/period controls: button surfaces with `aria-pressed`
- direct input mode: text input with validation

v3.6.12 does not convert the Time picker to radiogroup, spinbutton, native
`input[type="time"]`, or locale/timezone-aware behavior.

## Non-Goals

- `popover/` provider migration
- baseline `components.css` changes
- plugin or WordPress binding
- locale calendars or timezone behavior
- recurring event semantics
- mobile full-screen picker variant
- full range-selection a11y redesign beyond the current visual range model
