Hardening proof

Try the following on the glyphs above:

  • Double-click a glyph: selection should not happen (user-select: none).
  • Click-and-drag a glyph: nothing should drag out (draggable="false" + -webkit-user-drag: none).
  • Right-click a glyph: context menu should show page-level options, not "Copy text" / "Search for ...".
  • Translate the page via the browser's translate feature: glyphs should remain as Material Symbols glyphs, not as translated words (translate="no" + notranslate).
  • Click on a glyph inside an icon button: the click should land on the button, not the glyph (pointer-events: none on the glyph).
  • Use a screen reader on an icon button: reads the button's aria-label only; the glyph's ligature text is hidden via aria-hidden="true".

Forbidden scope — icon font must not appear inside .prose

Charter §5 + §6 + prose.css §12. Even if a Material Symbols glyph accidentally lands inside long-form prose (e.g. a block pattern that uses one inappropriately), the prose-side enforcement forces it back to ligature text rendering so the leak is obvious to the author.
