Focus the search bar below. It should elevate (level2 shadow + 1px lift) and a suggestions panel appears beneath it. Click a suggestion to populate the input. Press Esc once with text to clear; press again to collapse.

Allowlisted host — .search-bar

Standard .search-bar markup from components.css §10. This module adds the expansion behavior; the underlying structure is unchanged.

Keyboard + Escape policy

Test these flows on the search bar above:

  • Tab to focus → bar elevates, suggestions popup appears.
  • ↓ (ArrowDown) from input → focus moves to first suggestion.
  • ↓ / ↑ on suggestions → navigate the list.
  • ↑ at first suggestion → focus returns to input.
  • Home / End on suggestions → jump to first / last.
  • Enter on suggestion → populates input, collapses.
  • Type any text, then Esc → text clears, panel stays open.
  • Esc again (empty input) → blur + collapse.

Forbidden scope — search bar must not expand

Intake contract §1 forbids attaching the expansion JS to a .search-bar that has any of .prose, .wp-block-post-content, .entry-content, or [contenteditable] as an ancestor. The two search bars below should behave as plain HTML5 search inputs: no elevation change, no suggestions popup, no .is-search-active class.

.prose container (federation territory — no expansion here)

Reduced-motion path

Enable prefers-reduced-motion: reduce at the OS or DevTools level, then focus the allowlisted search bar above. The suggestions panel should appear instantly (no fade / scale animation), and the bar should not animate its translateY lift. Box-shadow elevation change is retained because shadows are depth indication, not motion.

JS-disabled fallback

Disable JavaScript in DevTools and reload. The allowlisted search bar should still:

  • elevate on focus (via the CSS :focus-within selector, no JS needed)
  • accept input normally
  • submit on Enter if wrapped in a <form>

The suggestions popup will not appear (it is built by JS), but the bar remains usable.
