Table

Ready PHP layer needed blocks.css §6

core/table은 <figure class="wp-block-table"> 안에 <table>이 들어있는 형태로 출력됩니다. blocks.css가 figure를 wrapper로 활용 (border, radius, overflow-x).

Default

WP core 기본 렌더링. 전체 셀 격자 테두리. thead / tbody / tfoot 모두 지원.

Row separators (is-style-row-separators)

수직 경계선 제거 — 수평 row 구분선만 남김. 데이터 밀도가 높거나 열 수가 많아 수직선이 시각적 노이즈가 될 때 사용.

Stripes (is-style-stripes)

Wide table — horizontal scroll (PHP wrapper needed)

셀에 white-space: nowrap이 있으면 table이 figure 밖으로 overflow됩니다. 스크롤 격리는 <div class="wp-table-scroll"> wrapper가 필요하며, WP core/table 블록에서는 render_block 필터로 주입해야 합니다. 현재 catalog는 PHP 미적용 상태 — overflow 동작을 그대로 노출합니다.

Wide table — wrap (is-style-wrap)

Opt OUT of horizontal scroll. Cells wrap to multiple lines. Use when table content is text-heavy (descriptions, notes).

With figcaption

core/table에서 caption 입력 시 <figure> 내부 <table> 다음 형제로 <figcaption> 출력. 테이블의 border/radius는 <table>이 소유하므로 caption은 테두리 밖에 놓입니다.

core/search contracts. Four button-position modifiers (button-outside, button-inside, no-button, button-only) × two styles (default outlined / is-style-filled-search) + icon button variant. blocks.css §10.

A. WP core/search — 4 button-position layouts

WP core/search는 본질적으로 <form>입니다 — label + input + submit. WP editor가 노출하는 4가지 layout 옵션이 있고, 각각 의도가 다릅니다. 같은 form, 다른 배치.

core/search 4 layouts
button-outside

Explicit submit. 폼이라는 점을 가장 또렷이 보여줌. 기본 default.

button-inside

Compact. 헤더처럼 가로 공간 제약이 있을 때.

no-button

Enter로만 제출. 자동 완성·실시간 검색과 조합.

button-only

검색 오버레이를 여는 trigger. 실제 input은 다른 곳에서.

A.1 With visible label

실제 form context에서는 label을 보이게 두는 편이 명확합니다 (검색 페이지의 hero 검색, 회원 가입 폼 등). showLabel: true.

core/search showLabel

Tip: 큰따옴표로 완전 일치 검색 가능. 예: "design tokens"

B. Aesthetic skins — 3 honest variants

동일한 form을 다른 surface 위에 어울리도록 입힌 시각 skin. 기능은 동일하고, 어디에 두느냐에 따라 골라 씀.

core/search 3 skins
Outlined (default)

본문 안에 inline. 1px outline pill. 가장 minimal.

Filled · pill

surface-container-high 위에 corner-full. Header / nav 영역에 자연스럽게 안착.

Soft · square

surface-container-low + corner-medium. 다른 form input과 같은 줄에 둘 때.
